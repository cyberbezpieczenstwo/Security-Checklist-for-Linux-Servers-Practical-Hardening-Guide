# Linux Server Security Checklist

Hardening a Linux server is not about installing dozens of security tools.

Most security incidents happen because of simple issues:

- unnecessary exposed services
- weak SSH configuration
- outdated packages
- excessive permissions
- missing monitoring

This repository collects practical security checks that can be applied to real Linux servers.

## Why this exists

Security guides often focus on advanced attacks, while basic configuration mistakes remain the most common problem.

This checklist focuses on the fundamentals.
